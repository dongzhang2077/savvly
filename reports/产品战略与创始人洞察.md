# Savvly 产品战略与创始人洞察

## 创始人背景

### 个人情况
```
基本信息：
├─ 年龄：29 岁
├─ 当前身份：温哥华 CS 学生 + 兼职工作者
├─ 职业经历：工作数年后投资学业
└─ 财务状态：消耗储蓄期（计划性负现金流）

财务情况：
├─ 收入：极度不规律（刚经历 3 个月失业，半月前恢复兼职）
├─ 支出：相对稳定（~$1,600 CAD/月 + 学费）
├─ 储蓄：工作期间积累，现用于投资学业
└─ 目标：控制消耗速度，撑到毕业 → 转为积累期

独特优势：
├─ 创始人即目标用户（深度同理心）
├─ 真实痛点验证（亲身体验，非猜测）
├─ 快速迭代能力（自己用自己的产品）
└─ 真实创始人故事（励志，易共鸣）
```

## 核心产品定位

### 主航道：不规则收入人群（保持不变）

**目标市场**：
- **主要用户**：25-35 岁不规则收入者
  - 自由职业者（"芙蕾雅"，28 岁平面设计师）
  - 零工经济工作者
  - 收入 $2,800-4,500/月（可变）
  - 试图建立应急基金和财务稳定

- **次要用户**：18-25 岁学生 + 兼职工作者
  - "学生山姆"，21 岁，收入 $1,500/月
  - 需要简单工具和财务教育

**市场规模与机会**：
- 44% 千禧一代从事副业/零工经济
- 68% 自由职业者报告收入不可预测困难
- 竞品空白：YNAB、Mint 等都假设固定收入
- Mint 关闭后，市场需要新的解决方案

**核心差异化**：
1. **不规则收入专家**（vs 竞品的固定薪资假设）
2. **焦虑缓解设计**（vs 竞品的警报/惩罚式设计）
3. **希望可视化**（vs 竞品的过去支出聚焦）

**差异化定位声明**：
> "Savvly 是为不可预测收入者设计的预算 App，提供焦虑缓解工具和希望驱动的可视化，在实现财务掌控的同时建立面向未来的信心。"

## 产品战略：双模式系统

### 为什么需要双模式？

**核心洞察**：不规则收入人群会经历多个生命周期阶段

```
典型用户旅程：
┌─────────────────────────────────────┐
│ 阶段 1：建立期                      │
│ ├─ 学生 + 兼职（创始人当前状态）    │
│ ├─ 初创公司早期（零/低工资）        │
│ ├─ 职业转型（辞职进修）             │
│ └─ 特点：可能负现金流，消耗储蓄    │
├─────────────────────────────────────┤
│ 阶段 2：成长期                      │
│ ├─ 自由职业者稳定期                │
│ ├─ 零工经济但收入渐稳               │
│ └─ 特点：收入覆盖支出，开始积累    │
├─────────────────────────────────────┤
│ 阶段 3：优化期                      │
│ ├─ 高收入自由职业者                │
│ ├─ 或：转回全职但保留副业          │
│ └─ 特点：优化税务、投资、财富增长  │
└─────────────────────────────────────┘

关键：同一个人会经历这 3 个阶段
→ Savvly 可以陪伴整个旅程
```

### 模式 1：积累模式（Accumulation Mode）- MVP 核心

**目标用户**：芙蕾雅（28 岁自由职业者）

**核心功能**：
1. **收入平滑可视化**
   - 显示收入峰谷与平均值对比
   - 将混乱可视化为可理解的模式

2. **最低月预算（Baseline Budget）**
   - 基于最低收入月设定安全预算
   - 用户只需设置一次"基础生存成本"
   - 高收入月自动处理超额部分

3. **税务预留（核心差异化！）**
   - 每笔收入自动标记 25-30% 为税务
   - 地区化税务估算（加拿大/美国不同）
   - 该部分不计入"可支配收入"
   - 季度税务提醒
   - 可扣除支出追踪（高级功能）

4. **应急基金目标追踪**
   - 3-6 个月目标突出显示
   - 里程碑庆祝（25%/50%/75%/100%）
   - 进度条可视化

5. **焦虑缓解设计**
   - 温和色彩系统（柔和蓝/绿/橙，避免刺眼红色）
   - 积极框架（"你有 $X 可用"vs"警告：超支"）
   - 非评判性语言（无羞耻/内疚策略）
   - "平静模式"选项（简化视图）

6. **希望可视化**
   - 未来预测："按此储蓄速度，6 个月后你将有 $X"
   - 目标时间线（"你在这里 → 你要去哪里"）
   - 小胜利放大（"这 $50 能覆盖你 2 周食品！"）

7. **智能分类与自动化**
   - AI 自动分类交易
   - 一键修正（系统持续学习）
   - 降低手动工作量 90%

8. **游戏化激励**
   - 每日签到连续
   - 徽章与成就
   - 进度条和里程碑

**心理框架**：
> "我需要从混乱走向稳定"

### 模式 2：跑道模式（Runway Mode）- MVP+1 功能

**目标用户**：
- 创始人当前状态（CS 学生消耗储蓄）
- 职业转型期（辞职进修、创业准备）
- 失业求职期（有储蓄缓冲）
- Gap Year 旅行者（计划性消耗）
- 早退休试验期（测试 FIRE 可行性）

**市场空白**：
- 现有 App 都假设用户"试图积累"或"不亏损"
- 但这个群体是**计划性负现金流**
- 他们不是失败者，而是投资者（投资人力资本/经历）

**核心功能**：

1. **储蓄跑道计算器**
```
公式：
储蓄跑道 = (总储蓄 - 预留大额支出) / 月净消耗

可视化：
┌────────────────────────────┐
│ 💰 储蓄跑道：12.5 个月     │
│ ████████████░░░░░░░░       │
│                            │
│ 你的目标：8 个月后毕业     │
│ 安全缓冲：+4.5 个月 ✓      │
│                            │
│ 按此速度，一切正常          │
└────────────────────────────┘

心理价值：
✓ 清楚知道"我还能撑多久"
✓ 验证"储蓄足够支撑到目标达成"
✓ 如果跑道缩短，触发行动提醒
```

2. **负现金流友好设计**
```
不再强调"储蓄率"
改为强调"消耗速度可控"

积极框架：
✓ "本月从储蓄提取 $1,200（计划内）"
✓ "你的计划在正轨上"
✓ "这是对未来的投资"

避免：
✗ "警告：负储蓄率！"
✗ "你每月亏损 $1,200"
✗ 红色警报、焦虑触发
```

3. **大额支出时间线**
```
学费/搬家/其他大额倒计时：
┌────────────────────────────┐
│ 📚 春季学费截止：45 天     │
│ 已准备：$3,200 / $5,000    │
│ ██████░░░░ 64%             │
│                            │
│ 每月需存：$400             │
│ （已从跑道计算中扣除）      │
└────────────────────────────┘

功能：
- 关联到储蓄跑道（自动扣除预留金额）
- 倒计时提醒
- 从储蓄预留（而非月度储蓄）
```

4. **未来收入预测**
```
┌────────────────────────────┐
│ 🎓 毕业后预期收入          │
│ $60,000 - $80,000/年       │
│                            │
│ 预计 8 个月后转为正现金流   │
│                            │
│ 可视化时间线：              │
│ 现在 ──────────┬───→ 毕业  │
│        消耗期  │   积累期   │
│   跑道：12月   └→ 缓冲：4月 │
└────────────────────────────┘

心理价值：
✓ 提供希望（不只是焦虑）
✓ 验证投资回报
✓ 连接"现在的痛苦"和"未来的收益"
```

5. **触发行动规则**
```
自动提醒：
├─ 跑道 < 10 个月：建议增加兼职时间
├─ 跑道 < 6 个月：警告需要调整
├─ 单月消耗 > 150% 平均：检查异常支出
└─ 大额支出 < 30 天：确认资金准备
```

6. **一键切换模式**
```
用户可以随时在两种模式间切换：
积累模式 ↔ 跑道模式

系统自动检测：
- 如果连续 3 个月负现金流
  → 建议："要切换到跑道模式吗？"
- 如果转为正现金流
  → 建议："恭喜！要切换到积累模式吗？"
```

**心理框架**：
> "我正在投资未来，这是战略性消耗"

## 创新功能：智能预算回滚（Budget Rollover）

### 问题：传统月度重置的缺陷

```
传统方式（YNAB、大多数 App）：
1 月：食品预算 $400，实际花 $350
→ 2 月：剩余 $50 消失，重新分配 $400

问题：
├─ "月底冲刺花钱"行为（"不花就浪费了"）
├─ 不符合真实生活（生活不按日历月重置）
├─ 惩罚节省行为（省钱 = 失去预算）
└─ 创造人为压力
```

### Emma（英国 App）的回滚方式

```
1 月：食品预算 $400，实际花 $350
→ 2 月：自动分配 $450（$400 + 上月剩余 $50）

优势：
✓ 奖励节省行为
✓ 自然的财务缓冲
✓ 减少"月底必须花完"压力
✓ 更符合真实生活模式
```

---

### Savvly 的实现：智能回滚（Smart Rollover）

**不是简单的"全部回滚"，而是类别级别可配置**：

#### 1. 完全回滚（Full Rollover）
```
适用：不规则支出类别
├─ 食品（有时多有时少）
├─ 交通（取决于接项目地点）
├─ 娱乐（可以攒几个月去旅行）
└─ 自由支配

行为：未用金额 100% 结转下月

例子：
1 月食品预算 $400，花 $350
→ 2 月食品自动分配 $450
```

#### 2. 部分回滚（Partial Rollover）
```
适用：固定支出但偶尔有波动
├─ 公用事业（冬天高夏天低）
└─ 通讯（偶尔超出流量）

行为：未用金额 50% 结转，50% 回到"可分配池"

例子：
1 月通讯预算 $80，花 $70
→ 2 月通讯分配 $85（$80 + $10 × 50%）
→ 可分配池 +$5
```

#### 3. 无回滚（No Rollover）
```
适用：严格固定支出
├─ 房租（每月固定）
└─ 订阅（每月固定）

行为：未用金额 100% 回到"可分配"池

例子：
1 月房租预算 $1,200，实际 $1,200
→ 2 月房租重新分配 $1,200
```

#### 4. 目标导向回滚（Goal Rollover）
```
适用：储蓄目标类别
├─ 应急基金
├─ 税务预留
└─ 大额购买（旅行、电脑等）

行为：未用金额永久累积，直到目标达成

例子：
1 月应急基金分配 $300，已有 $2,700
→ 2 月显示："应急基金 $3,000（目标 $6,000）50% ✓"
→ 继续累积直到达成 $6,000
```

└─ 自由职业者建议预留：25-35%

├─ 联邦税：10% - 37%
├─ 州税：0% - 13%（加州最高）

├─ Medicare：2.9%

## MVP 开发优先级

### Phase 1：核心 MVP（0-3 个月）

**目标**：服务"芙蕾雅"（主力用户），构建核心积累模式

**必备功能**：

1. **用户认证与入职**
   - 注册/登录
   - 入职流程（地区、收入、税务比例设定）
   - .edu 邮箱验证（学生免费）

2. **银行连接（手动优先，自动可选）**
   - 手动添加账户和交易
   - Plaid 集成（如时间允许）

3. **智能分类系统**
   - 预设分类（住房、食品、交通等）
   - 自定义分类
   - AI 自动分类（规则匹配 + 学习）
   - 一键修正

4. **不规则收入核心工具**
   - 收入可视化（简单图表）
   - 税务自动预留（28% 默认，可调整）
   - 最低月预算计算器（基于历史数据）

5. **预算管理**
   - 月度预算设定
   - 预算 vs 实际支出对比
   - 预算回滚功能（类别级别可配置）
   - 类别间转移（"随机应变"）

6. **进展驱动仪表板**
   - "可用收入"突出显示（扣税后）
   - 本月预算状态（温和色彩）
   - 应急基金进度条
   - 即将到来的账单

7. **目标追踪**
   - 应急基金目标（3-6 个月）
   - 自定义储蓄目标
   - 里程碑庆祝（25%/50%/75%/100%）

8. **基础报告**
   - 月度支出总结
   - 收入 vs 支出趋势
   - 分类支出饼图

### Phase 1.5：创始人自用增强（与 MVP 并行）

**目标**：让创始人能立即使用 Savvly

**快速增强（总计 5-7 天开发）**：

1. **储蓄跑道小部件（1-2 天）**
```
在仪表板增加卡片：
┌────────────────────────────┐
│ 💰 储蓄跑道                │
│                            │
│ 12.5 个月                  │
│ ████████████░░░░░░░░       │
│                            │
│ 基于当前消耗速度           │
│ [查看详情]                 │
└────────────────────────────┘

计算逻辑：
(总余额 - 用户设定的"预留金额") / 近 3 月平均净支出
```

2. **大额支出倒计时（1 天）**
```
目标功能的变体：
┌────────────────────────────┐
│ 📚 春季学费                │
│ 截止：45 天                │
│                            │
│ $3,200 / $5,000            │
│ ██████░░░░ 64%             │
│                            │
│ [标记为已支付]             │
└────────────────────────────┘

关联到储蓄跑道（自动扣除预留金额）
```

3. **负现金流友好文案（0.5 天）**
```
当检测到"收入 < 支出"时：
不显示：❌ "警告：超支 $1,200"
改显示：✓ "本月从储蓄提取 $1,200（计划内）"

在跑道小部件中显示：
"按此速度，你的储蓄可支撑 12 个月"
```

4. **预算回滚开关（2-3 天）**
```
类别设置中增加回滚策略选择：
- 完全回滚（默认推荐）
- 部分回滚（50%）
- 无回滚
- 目标导向（储蓄类别）

月底自动执行回滚逻辑
```

**价值**：
- ✓ 创始人立即成为产品的真实用户
- ✓ 快速发现问题并迭代
- ✓ 验证"跑道模式"需求
- ✓ 为未来正式功能打基础
